# Embedded Systems Career Development Journal

## 简介

嵌入式系统（英语：Embedded system），是一种嵌入机械或电气系统内部、具有专一功能和实时计算性能的计算机系统。嵌入式系统常被用于高效控制许多常见设备，被嵌入的系统通常是包含数字硬件和机械部件的完整设备，例如汽车的防锁死刹车系统。相反，通用计算机如个人电脑则设计灵活，可以智能处理各式各样的运算情况，以满足广大终端用户不同的需要。

从历史上的阿波罗登月计划，到现代SpaceX可回收火箭的成功，再到从手机电源适配器到最新亮相的东风61洲际导弹，嵌入式系统无处不在。理论的证明与推导固然是成功的基础，但这些设备的实现其实都依赖于嵌入式设备提供的高效、实时的计算性能。

## 学习路线

从实际工作的角度来说，嵌入式工程师并非分为嵌入式硬件工程师和嵌入式软件工程师，如果想成为的是嵌入式软件工程师，就要先抛弃掉硬件，专攻嵌入式软件方面，这并不是说硬件工程师不行，而是硬件有专门的人负责。

学习硬件方面的知识固然有助于成为一个全面的嵌入式专家，但如果你是刚刚入门的菜鸟级别工程师（基础的C语言都没有学到指针那种），强行学习硬件和软件只会让你“消化不良”，并且大概率都学得不深入。这样就只需要看懂原理图，懂得简单的电路分析方法：基尔霍夫定律、戴维南定律、节点网孔法就足够应付初级工程师在工作中涉及到的硬件了。

学好C语言对于嵌入式工程师而言是最基础的，毕竟C语言诞生就是为了编写计算机操作系统（最根本是为了移植电子游戏），而计算机操作系统就是对计算机的硬件资源进行调度的一种程序，因而C语言和计算机硬件、操作系统的关联程度比其他语言都要更加紧密。但是同样的，如果不整体了解计算机体系的全貌以及具体工作中可能会遇到的问题，新手真的很难理解一些语言特性的背后藏着怎么样的思考，因此C语言不需要太刻意去学，通过大量的写代码在实践中学习C语言，这样理解才是最为深刻也是最高效的。

C语言指针之前的内容（包括指针）学会了之后，就可以找一块MCU开发板学起来了，对于入门的学生/其余行业的工程师而言，最常见的嵌入式平台就是STC89C51、STm32、ESP8266、ESP32。即使你的C语言基础很不错，我还是建议你先从最简单的51单片机开始学习，熟悉寄存器、定时器、外设等概念，可以看B站的Up主<江协科技>来入门学习C51单片机，51单片机是最经典的8位单片机，补充阅读可以选择《手把手教你学51单片机》。

基于 STM32 的嵌入式软件开发大体可以分为两种主流方式：基于标准外设库（Standard Peripheral Library, SPL）的开发以及基于 HAL 库（Hardware Abstraction Layer, HAL）的开发。

标准外设库在很大程度上接近底层寄存器结构，开发过程中需要对芯片手册、寄存器位定义以及总线结构有较为深入的理解，因此对新手而言上手较为晦涩。但正因其对寄存器控制较为直接，更有助于掌握微控制器的底层运行机制和外设工作原理。

HAL 库（Hardware Abstraction Layer，硬件抽象层）是 ST 公司提出的统一软件框架，通常配合图形化配置工具 STM32CubeMX 使用。通过 CubeMX 可对时钟树、外设、引脚复用及中断等进行可视化配置，并自动生成初始化代码和工程模板，从而降低开发门槛并缩短开发周期。

但HAL因为封装较厚，HAL 在代码体积、执行效率和灵活性方面通常不及基于标准库或寄存器级的开发方式。网上关于标准外设库和HAL库的争论一直很多，任何被广泛使用的方案都是有其存在的原因的，目前单片机的性能一路狂飙到GHz级别，高封装性、高抽象性带来的开发便捷与可移植性为超大型项目提供了可能
